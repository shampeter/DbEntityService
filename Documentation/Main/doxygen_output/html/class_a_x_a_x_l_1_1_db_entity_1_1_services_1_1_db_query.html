IQuery<T> AXAXL.DbEntity.Services.DbQuery< T >.InnerJoinOr< TParent, TChild > ( params Expression< Func< TChild, bool >>[]  orClauses)

A group of conditions evaluated together by an "OR" operator. This and other conditions supplied by Where(Expression<Func<T, bool>>) and And(Expression<Func<T, bool>>) will be evaluated together by "AND" sql operators.

For example, in a parent -> child -> grand-child relationship. The following code

InnerJoinOr<child, grand-child>( grandchild -> grandchild.SomeProp1 = SomeValue1, grandchild.SomeProp2 == SomeValue2)

will be executed as

FROM parent t0
INNER JOIN child t1 on t0.primary_key = t1.forign_key
INNER JOIN grant-child t2 on t1.primary_key = t2.foreign_key
WHERE t2.some_prop1 = somevalue1 or t2.some_prop2 = somevalue2

IQuery<T> AXAXL.DbEntity.Services.DbQuery< T >.LeftOuterJoinOr< TParent, TChild > ( params Expression< Func< TChild, bool >>[]  orClauses)

A group of conditions evaluated together by an "OR" operator. This and other conditions supplied by Where(Expression<Func<T, bool>>) and And(Expression<Func<T, bool>>) will be evaluated together by "AND" sql operators. Left out join is assumed between TParent and TChild .

For example, in a parent -> child -> grand-child relationship. The following code

LeftOuterJoinOr<child, grand-child>( grandchild -> grandchild.SomeProp1 = SomeValue1, grandchild.SomeProp2 == SomeValue2)

will be executed as

FROM parent t0
LEFT OUTER JOIN child t1 on t0.primary_key = t1.forign_key
LEFT OUTER JOIN grant-child t2 on t1.primary_key = t2.foreign_key
WHERE t2.some_prop1 = somevalue1 or t2.some_prop2 = somevalue2
